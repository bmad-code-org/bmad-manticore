The tools, the scripts, and the film strip that turn talking into a finished video.

Taste lives in files. Mechanics live in scripts, run through uv. Skills are thin routers between them, so smaller models can drive the pipeline and it gets cheaper over time without getting worse.

Architecture

One stage, every time

Every stage skill runs the same loop, no creativity in the mechanics. It loads config, reads state, runs a script, checks a checklist, writes the artifact, and stops dead at a gate.

The studio config layers three ways: packaged defaults, then team overrides, then personal ones. The four gates (outline, cutplan, beats, final) are the only places a stage stops, and only your explicit say-so moves one from pending to a date.

Flow  ·  the writing lane

Spoken idea to script to re-record

The interview-recording trick: answer the brain-dump questions on camera, each prefixed with the cue "question from the interviewer", and the cut stage segments it mechanically. The interview itself becomes usable footage, not just notes.

Flow  ·  the edit lane

Raw footage to awesome

Footage-first projects (a livestream VOD, a recorded talk, raw takes) enter at the cut and run the same gates from there. The editor timeline (edl.json, cutplan.md, FCPXML) is written alongside every iteration, so you can finish in your own editor at any step.

Generation, honestly

Where synthesis fits, and where it must not

Voice + sound (mc-audio)

  • Local and free: Kokoro-82M narration and two-host dialogue, MusicGen beds, AudioLDM2 whooshes and chimes
  • Honest limits, always stated: stock voices, no cloning; narration in YOUR voice means recording it
  • Songs with vocals: planned (ACE-Step), never promised until validated

Generated b-roll

  • Atmosphere and story-beat clips through your configured lanes, today
  • Real verified imagery is preferred first; generation fills only what does not exist
  • A hand-built text card is the last resort, not the first

The hard safety rules

  • Generated footage never depicts UI or text that must read correctly; that comes from screen recordings
  • Never chain generative edits: a revision of a revision degrades like a photocopy of a photocopy
  • Every tweak starts clean: all the original assets again, one improved prompt

Small deterministic fixes (a logo swap, one wrong line, a color correction) are composited programmatically with ffmpeg or rsvg, never regenerated. No metered lane ever runs without your explicit setup choice.

The render pipeline

One EDL, two renders, every editor

The FCPXML timeline exports on every cut approval (Resolve and Final Cut import it natively); xmeml (Premiere) and CMX3600 EDL are planned, so those users work from edl.json, cutplan.md, and the renders meanwhile. Same approved EDL, same composite core, so the preview you signed off on is exactly what the final bakes.

Graphics engines

The beat table is the contract

One engine-neutral table, produced at gate 3, consumed by every engine. Each row anchors a graphic to the exact word it lands on in the edited timeline. Before the table exists, Manny riffs the ideas with you: he pitches the moments and treatments he sees, you say what you were picturing, and the mix (cards, diagrams, imagery, clips, memes) follows your Production Bible, not a default.

HyperFrames

Default per-video overlay beats. Registry checked before authoring; exports ProRes 4444 alpha.

Remotion

Stingers, transitions, React-stateful graphics, shorts karaoke captions. Dual render for OBS and the editor.

OGraf

Editable broadcast graphics, only where the target supports it (Resolve 21+, or OBS live). Everyone else gets baked alpha.

HTML lane

Author from scratch, rendered to exact pixels by Playwright, then frame-verified over a checkerboard for alpha.

Every final render is checked with render_verify (pixfmt, duration, fps, resolution, extracted frames) before it is called done. The design-prompting loop drives a capable model when a beat needs bespoke animation nothing in the registry fits.

Configuration + extensibility

Your studio, in plain files

The [[tools]] registry

  • Register any generation CLI with a headless invocation and a notes field
  • The notes are persistent memory: model quirks, what it is bad at, how output lands
  • Skills drive your CLIs right every session instead of rediscovering them

Taste contracts

  • The Production Bible: density, image-type policy, overlay aesthetic, CTA inventory
  • The voice bible: how you talk, evidence-cited from your own transcripts
  • The blacklist: LLM tells and phrases you never say, enforced by the linter

Extend without code

  • Per-skill customize.toml overrides, merged team then personal
  • Formats are plain markdown files: a new format is a new file, not new code
  • Each format picks which stages run and carries its own accumulated learnings

The compounding loop

It gets smarter every video

Retro routes each post-publish note to the file that would have prevented it: voice bible, Production Bible, blacklist, format learnings, tool notes. The ratchet only turns one way, and it never weakens a gate.
